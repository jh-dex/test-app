# Live Board MVP Draft

피그마 느낌의 **웹 기반 실시간 작업 보드 MVP 초안**입니다.

## 포함 기능 (MVP)

- 실시간 스케치(펜/지우개)
- 이미지 업로드 후 보드에 배치
- 드래그로 이미지 이동
- 같은 브라우저 여러 탭 + 다른 PC 간 실시간 동기화(SSE/HTTP + BroadcastChannel)
- 간단한 참여자(presence) 표시

> 참고: 이제 기본 포함된 Node 동기화 서버(`server.js`)를 통해 같은 네트워크의 다른 PC에서도 실시간 협업이 가능합니다.

## 실행 방법

정적 파일이라서 아무 간단한 서버로 실행하면 됩니다.

```bash
npm install
npm start
```

브라우저에서 `http://localhost:4173` 접속 후,
같은 주소를 다른 PC에서도 열면 실시간으로 같은 보드를 공유합니다.

같은 보드에 초대하려면 **같은 room URL**을 공유하세요.

- 예: `http://localhost:4173/?room=team-design`

## 다음 단계 제안

1. 방(Room) 개념 및 초대 링크(예: `/room/abc123`)
2. 오브젝트 선택/리사이즈/회전 고도화
3. 보드 저장/불러오기(스토리지/DB 연동)
4. 권한(읽기/쓰기), 인증, 접속 제어

## 간단 저장(현재 구현)

- 서버가 각 room의 이벤트 히스토리를 `data/rooms.json`에 자동 저장합니다.
- 서버 재시작 후에도 해당 파일이 남아 있으면, 같은 `?room=...` 접속 시 기존 상태가 복원됩니다.
- 완전한 운영용 저장소(DB) 대신, 로컬 파일 기반의 간단한 지속성(persistence)입니다.

## 품질 관련 참고

- 스케치 레이어를 SVG 스트로크 기반으로 구성해 확대 시에도 선명도를 유지합니다.
- 스트로크는 포인트 시퀀스를 곡선(path)으로 렌더링해 직선 분절감을 줄였습니다.
- 지우개는 스케치 레이어에서 배경색 스트로크 방식으로 동작합니다.

## 배포/서버 필요 여부

요약하면 **"완전 정적 호스팅만으로는 다른 PC 실시간 공유가 안 됩니다."**

- 같은 브라우저 탭끼리만: `BroadcastChannel`만으로 동작 (서버 불필요)
- 다른 PC/다른 기기까지: 동기화 이벤트를 중계할 **서버 프로세스가 필요**

현재 프로젝트는 `server.js`가 그 역할을 하므로,
동일한 사이트(같은 도메인)에서 `node server.js`로 함께 띄우면 바로 반영됩니다.
즉, Figma처럼 "사이트 접속만으로 바로 공유"를 하려면,
결국 백그라운드에서 항상 떠 있는 서버(직접 운영 or PaaS 배포)가 필요합니다.

### 정적 호스팅에 프론트만 올린 경우

- `index.html/app.js`만 CDN/정적 호스팅에 올리면, 다른 PC 실시간 동기화는 동작하지 않습니다.
- 이때는 별도 동기화 서버 주소를 `?sync=` 쿼리로 지정할 수 있습니다.
  - 예: `https://your-site.example/?sync=https://sync.example.com`

### 자주 묻는 질문 (GitHub만 올리면 되나요?)

- **GitHub 리포에 코드만 push**: ❌ 자동 실시간 동기화 안 됨
  - 이유: GitHub 자체는 이 프로젝트의 `server.js` 런타임을 계속 실행해주지 않음
- **GitHub Pages(정적 호스팅)**: ❌ 다른 PC 실시간 공유 안 됨
- **Node 서버가 실행되는 배포(Render/Railway/Fly.io/EC2 등)**: ✅ 실시간 공유 가능

즉, 사용자 입장에서는 사이트만 열면 되지만, 서비스 운영 측면에서는
`server.js`가 항상 실행 중인 환경이 필요합니다.
